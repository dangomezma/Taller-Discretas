2. ¿Qué idea matemática usa?
R\\ Usa la idea de comparticion de secretos aditiva con aritmetica modular
cada nota x se divide en tres partes s1,s2,s3 generadas de forma que x ≡ s1+s2+s3 mod M
s1 y s2 se generan completamente al azar y la tercera (s3) se calcula como lo que falta para que la suma module a x: s3 = x - s1 -s2
gracias a que s1 y s2 son generados al azar estos no dan ninguna informacion sobre x, parece un simple numero entre 0 y M
y al sumar estas 3 partes modulo M da la nota original. Esta propiedad se puede extender a listas de tal forma que al sumar todas las partes que tiene cada servidor y luego sumando esos 3 resultados
se reconstruye la suma total sin que ningun servidor haya visto una nota completa

6. Ejemplo de cómo una nota x se escribe como s1 + s2 + s3 (mod M) sin revelar x
R\\ Supongamos que un estudiante tiene una nota x = 40, y usamos un modulo pequeño para el ejemplo, digamos M = 100 (en el programa real se usa M = 1000003, pero la idea es la misma)
Se generan dos números aleatorios, por ejemplo s1 = 63 y s2 = 22. Luego se calcula la tercera parte para que la suma module correctamente a x:
s3 = (x - s1 - s2) mod M
s3 = (40 - 63 - 22) mod 100
s3 = (-45) mod 100
s3 = 55
Ahora el servidor 1 tiene 63, el servidor 2 tiene 22, y el servidor 3 tiene 55. Ninguno de estos tres números, visto de forma aislada, se parece a 40 ni da ninguna pista sobre su valor — podrían corresponder a cualquier nota posible, 
ya que s1 y s2 fueron elegidos completamente al azar. Solo al sumar los tres y aplicar el módulo se recupera el valor original:
(63 + 22 + 55) mod 100 = 140 mod 100 = 40

Ejercicio_9
1. ¿Qué problema resuelve el programa?
R\\ El programa mide cuanta informacion (o incertidumbre) contiene un texto usando la entropia de Shannon. Calcula la frecuencia y la probabilidad de aparicion de cada simbolo
Y a partir de esa probabilidad obtiene un numero que la representa, en promedio, cuantos bits se necesita para representar cada simbolo si se codificara de manera optima
Tambien permite comparar la entropia de dos textos distintos para determinar cual es mas "impredecible". 
Como extencion tambien se implemento codificacion de Huffman, un metodo real de comprension que aprovecha esta idea para asignar codigos mas cortos a los simbolos mas frecuentes
2. ¿Qué idea matemática usa?
R\\ Usa la formula de entropia de Shannon: H = -Σ pi log2(pi) donde pi es la probabilidad de cada simbolo distinto en el texto
Esta formula funciona de tal manera que un simbolo que se repite mucho aporta poca informacion nueva cuando aparece, en cambio un simbolo raro aporta mas informacion
esto gracias al log2(pi) ya que entre mas pequeña sea la probabilidad, mas negativa y mas grande en magnitud es su logaritmo
Para la extencion se usa un arbol de Huffman: un algoritmo que construye un arbol binario uniendo repetidamentelos dos simbolos menos frecuentes, hasta formar un solo arbol
Recorre ese arbol desde la raiz hasta cada simbolo marcando 0 o 1 si va a la izquierda o derecha lo que genera un codigo binario para cada simbolo, donde los simbolos mas frecuentes
terminan con codigos mas cortos porque quedan mas cerca de la raiz
3. ¿Cómo se ejecuta?
R\\ ...
4. ¿Qué pruebas hicieron?
R\\ ...
5. ¿Qué limitaciones tiene la solución?
R\\ El calculo de la entropia trata cada simbolo de forma completamente independiente, no considera el contenido ni la relacion entre simbolos consecutivos
por lo que un texto con patrones predecibles pero muchos simbolos distintos como "ababababa" podria dar una entropia similar a un texto sin ningun patron
6. ¿Por qué la entropía mide incertidumbre y no simplemente longitud del texto?
R\\ La entropia no depende de cuantos caracteres tiene el texto, sino de que tan predecibles son los caracateres
un texto largo pero repetitivo como "aaaaaaaaaaaaa" tiene seguramente entropia 0 o cercana a esta, porque una vez que sabes el simbolo "a", siempre vas a acertar
En cambio un texto mas corto y sin ningun patron como "qoiwhe" tiene una entropia alta porque cada nuevo caracter es practicamente imposible de adivinar antes de verlo

R\\ Un qubit se representa como un vector de dos entradas [α, β] donde α y β son numeros asociados a los estados |0⟩ y |1⟩ 
Aplicar una compuerta cuantica equivale a multiplicar una matriz 2x2 por ese vector
Las probabilidades de medir cada resultado se obtienen elevando al cuadrado cada componente del vector |α|² y |β|² las cuales siempre suman 1 ya que representan la distribucion de probabilidad completa
Finalmente para simular las mil mediciones se usa muestreo aleatorio ponderado: se generan 1000 valores entre 0 y 1 siguiendo esas probabilidades calculadas,
imitando el hecho de que en mecanica cuantica el resultado de medir un qubit es intrinsecamente probabilistico, no determinista

5. ¿Qué limitaciones tiene la solución?

6. ¿Diferencia entre probabilidad cuántica simulada y ejecución en un computador cuántico real?
R\\ En este programa las "probabilidades cuanticas" son en realidad numeros calculados con algebra lineal basica 
y las mil mediciones se generan con un generador de numeros pseudoaleatorios corriendo en una computadora normal
Es decir, todo este proceso es completamente determinista y transparente: se sabe desde un principio cuales son las probabilidades exactas porque se calculan explicitamente
y el azar viene unicamente del algoritmo de numeros pseudoaleatorios.
En un computador cuantico real, el qubit es un sistema fisico real (como un electron, foton o un circuito superconductor) cuyo estado evoluciona segun las leyes de la mecanica cuantica
Al aplicar una compuerta, se manipula fisicamente ese sistema y al medirlo ese resultado es intrinsecamente aleatorio por la naturaleza misma de la fisica cuantica
Es una incertidumbre fundamental del universo, que ademas esta sujeta a ruido fisico real(interferecias de entorno) que pueden alterar los resultados
En conclusion: este programa reproduce predicciones matematicas de la teoria cuantica de forma exacta, pero no reproduce el mecanismo fisico real detras de esas predicciones, ni sus imperfecciones practicas
